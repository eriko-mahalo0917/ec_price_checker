# rakuten_api.py メモ

## ■ 実装目的

楽天API（商品検索API）を利用して商品検索を行い、
商品名・価格・店舗・URL・レビュー情報を取得する

---

## ■ 躓いたポイントと解決

### ① sysが未定義エラー

【エラー】
NameError: name 'sys' is not defined

【原因】
sysをimportしていなかった

【対応】
import sys を追加

---

### ② dotenvがimportできない

【エラー】
ModuleNotFoundError: No module named 'dotenv'

【原因】
python-dotenvが未インストール

【対応】
pip install python-dotenv

---

### ③ 401エラー（認証エラー）

【原因】
.envの環境変数名が不一致
（RAKUTEN_API_ID と RAKUTEN_APP_ID のズレ）

【対応】
.envとコードの変数名を統一

---

### ④ 400エラー（リクエスト不正）※重要

【原因】
以下の複合要因

・楽天APIのエンドポイントURLが変更されていた  
・アクセスキー（accessKey）が必須になっていた  
・パラメータ不足によりリクエスト不正となっていた  

【対応】
・最新のAPI仕様に合わせてURLを修正  
・applicationId に加えて accessKey を追加  
・パラメータを正しく設定  

```python
params = {
    "applicationId": os.getenv("RAKUTEN_API_ID"),
    "accessKey": os.getenv("RAKUTEN_ACCESS_KEY"),
    "keyword": product_name,
    "format": "json",
    "hits": 30,
    "page": page
}
````

---

### ⑤ IPアドレスによるエラー

【原因】
テザリング環境で実行していたため、
楽天API側で制限または異常リクエストと判断された可能性

【対応】
通常のネットワーク環境で実行したところ正常動作

---

## ■ 学び

・APIは「認証情報（applicationId / accessKey）」に強く依存する
・API仕様（URL・必須パラメータ）は変更される可能性がある
・400エラーは「パラメータ不備」の可能性が高い
・401エラーは「認証情報ミス」の可能性が高い
・環境（IP・ネットワーク）もエラー原因になる
・ログ出力やprintでの確認が重要
・API仕様変更により既存コードが動作しなくなる可能性があるため、定期的な仕様確認が必要

# price_list_builder.py メモ (2026-03-21 実装)

## ■ 実装目的
APIから取得・整形した商品データのリストを受け取り、
「除外フィルタリング」および「価格順の並び替え」を行う。

---

## ■ 設計のこだわり
### ① 処理の順番
「並び替え」の前に「除外（フィルタリング）」を行う。
・不要なデータを先に捨てることで、並び替えの計算量を減らす。
・中古品などが最安値として上位に来るのを防ぐ。

### ② メソッドの汎用性
引数名を `product_data_list` で統一。
「フィルター済み」でも「未処理」でも、商品リストの形（辞書のリスト）であれば
何でも処理できる「万能な道具」として設計した。

---

## ■ 躓いたポイント・学び

### 【重要】リスト内包表記によるフィルタリング
中古品や訳あり品を除外する際、Python特有の「リスト内包表記」と「any関数」を組み合わせて実装。

```python
filtered_list = [
    product for product in product_data_list
    if not any(keyword.lower() in product["name"].lower() for keyword in EXCLUDE_KEYWORDS)
]

---

### ■ リスト内包表記の1行詳細解説

* **`product`**
    * **役割**: 新しいリストへの「追加実行」
    * **解説**: 条件をすべてクリアした「合格品」を、最終的な新しいリストの中に放り込みます。
* **`for product in product_data_list`**
    * **役割**: データの「取り出しループ」
    * **解説**: 元のリスト（全データ）から、商品を1つずつ取り出して `product` という名前で検査台（判定処理）へ運びます。
* **`if not any(...)`**
    * **役割**: 「不合格者」のシャットアウト
    * **解説**: 「もし、以下のNGワードに **1つも（any）** 当てはまら **ない（not）** なら合格」という判定フィルターです。
* **`keyword.lower() in product["name"].lower()`**
    * **役割**: 文字列の「部分一致チェック」
    * **解説**: 除外ワードが商品名の中に含まれているかを探します。`.lower()` を使うことで、大文字の "USED" と小文字の "used" などの表記揺れを無視して確実にヒットさせます。
* **`for keyword in EXCLUDE_KEYWORDS`**
    * **役割**: NGワードの「巡回」
    * **解説**: 設定ファイルにある複数のNGワード（中古、訳あり等）を、1つずつ順番にスキャンして試します。
